# MCU LootBox 嵌入式项目开箱生成器

MCU LootBox 是一个纯前端的嵌入式项目灵感生成工具。它用原创的“横向滚动开箱 + 中央指针 + 减速停靠”交互方式，从主控、传感器、显示、通信、存储、执行器等硬件池中随机组合出一个嵌入式项目方案。

本项目不使用 CS2、Valve、Steam 的任何官方图片、Logo、字体、音效或受版权保护素材，只参考“开箱抽奖”的交互形式。

## 项目简介

很多嵌入式项目的难点不是写第一行代码，而是先想清楚“用什么硬件组合做一个什么项目”。MCU LootBox 试图把这个过程变得更直观：用户点击一次抽奖，页面会生成多个硬件模块的抽取动画，并自动整理成项目名称、简介、BOM、兼容性分析、接线建议和开发提示词。

它适合用作课程设计选题、开源项目灵感生成、硬件组合评估、README 草稿生成和嵌入式学习练习。

## 功能亮点

- 多模块同步开箱：多个硬件轨道同时滚动，并按顺序依次停下。
- 动态模块槽位：默认 6 个模块，可额外添加传感器、显示、通信、存储、执行器等同类模块。
- 单模块重抽：每个模块最多可重抽 3 次。
- 稀有度系统：蓝色、粉色、金色三档硬件池，保留概率抽取机制。
- 项目方向模式：支持环境监测、姿态检测、智能小车、数据记录仪、蓝牙遥控、物联网监测等方向偏好。
- 真实可做优先模式：尽量减少低性能主控搭配高功耗、高复杂模块的情况。
- 硬件图鉴：可查看完整奖池，支持搜索和稀有度筛选。
- 自动生成项目表、BOM、兼容性分析、STM32F103C8T6 接线建议。
- 支持复制 Markdown、生成 Codex 开发提示词、生成完整 GitHub README 草稿。
- 支持 JSON 导出/导入和浏览器 localStorage 历史收藏。
- 使用 Web Audio API 生成原创开箱音效，不依赖外部音频文件。
- 无需后端，直接用浏览器打开 `index.html` 即可运行。

## GitHub Pages 部署说明

1. 打开你的 GitHub 仓库页面。
2. 进入仓库的 `Settings`。
3. 在左侧菜单中找到 `Pages`。
4. 在 `Source` 中选择 `Deploy from a branch`。
5. 在 `Branch` 中选择 `main`。
6. 在 `Folder` 中选择 `/root`。
7. 点击保存后，等待 GitHub Pages 自动部署完成。
8. 部署完成后，GitHub 会生成一个在线访问链接。
9. 将生成的在线链接填入本 README 的“在线演示”区域。

## 使用方法

1. 下载或克隆本项目。
2. 用浏览器直接打开 `index.html`。
3. 选择项目方向模式和抽取策略。
4. 可选：在模块轨道中点击“+ 添加同类模块”增加额外模块。
5. 点击“开始抽奖”。
6. 等待所有模块依次停下。
7. 查看生成的项目表、BOM、兼容性分析和接线建议。
8. 可对单个模块进行重抽。
9. 可复制 Markdown、生成 Codex 提示词、生成完整 README 或导出 JSON。

## 抽奖规则

每个模块轨道会生成 40 到 60 张硬件卡片，并预先确定一个中奖硬件。动画开始后卡片从右向左滚动，经过减速后，中奖硬件会精准停在中央指针下方。

整体抽奖时，当前页面中有多少个模块槽位，就会有多少条轨道参与抽奖。默认有 6 个模块：

- 主控核心
- 感知模块
- 显示交互
- 通信模块
- 存储模块
- 执行器/输出模块

## 稀有度说明

| 稀有度 | 类型 | 默认概率 |
| --- | --- | --- |
| 蓝色 | 普通入门 | 65% |
| 粉色 | 进阶实用 | 28% |
| 金色 | 高级展示 | 7% |

稀有度影响硬件池来源、项目评分和视觉效果。项目方向模式只会对相关硬件加权，不会取消稀有度系统。

## 模块配置说明

默认主控核心只能有 1 个，不能额外添加。以下模块可以增加同类槽位：

- 感知模块
- 显示交互
- 通信模块
- 存储模块
- 执行器/输出模块

限制规则：

- 每类模块最多 3 个。
- 总模块数最多 10 个。
- 抽奖过程中不能添加或删除模块。
- 新增模块可以删除，默认模块不能删除。
- 删除新增模块后，同类模块会自动重新编号。

例如添加两个显示模块后，会出现：

```text
显示交互
显示交互 2
显示交互 3
```

## 单模块重抽说明

整体抽奖完成后，每个模块都可以单独重抽。单模块重抽只会滚动当前模块轨道，其他模块保持不变。

规则：

- 每个模块最多重抽 3 次。
- 重抽期间会禁用所有抽奖相关按钮，避免并发动画导致状态错乱。
- 重抽完成后，项目表、BOM、兼容性分析、接线建议、Markdown、README 和 JSON 数据会基于新的硬件组合刷新。

## 奖池一览说明

页面提供“奖池一览 / 硬件图鉴”模块，用于查看当前抽奖使用的全部硬件池。

支持：

- 按模块分类查看硬件。
- 按蓝色、粉色、金色稀有度分组。
- 搜索硬件名称、接口、电压、备注等字段。
- 只看蓝色、粉色或金色硬件。
- 查看当前硬件总数和各稀有度数量。

奖池一览直接读取 `script.js` 中当前抽奖使用的数据，不单独维护重复硬件列表。

## BOM、兼容性分析、STM32 接线建议

抽奖完成后，页面会自动生成：

- 项目名称
- 项目简介
- 难度等级
- 含金量评分
- 推荐应用方向
- 后续扩展建议
- BOM 清单
- 兼容性分析
- STM32F103C8T6 推荐接线表
- 项目开发路线图

兼容性分析会尝试检查：

- 3.3V 主控与 5V 模块的电平问题。
- I2C、SPI、UART 等接口占用情况。
- 多个 SPI 设备是否需要独立 CS。
- 高功耗模块是否需要额外供电。
- 多执行器、多显示、多传感器带来的复杂度变化。
- 新手是否适合直接制作该组合。

当主控为 `STM32F103C8T6` 时，会自动生成推荐接线表，例如 I2C、UART、SPI、ADC、PWM、GPIO 的默认引脚分配。

## Markdown / README / JSON 导出

项目生成后支持：

- 复制项目表为 Markdown。
- 复制 BOM 为 Markdown 表格。
- 复制开发路线图为 Markdown。
- 生成 Codex 开发提示词。
- 生成完整 GitHub README 草稿。
- 导出当前项目为 JSON。
- 导入 JSON 恢复项目方案。
- 收藏项目到浏览器 localStorage 历史记录。

JSON 会尽量保存当前模块槽位、硬件结果、BOM、兼容性分析、评分、难度和生成时间。

## 免责声明

本项目是学习和创意生成工具，生成的硬件组合仅作为项目灵感和初步评估参考，不代表可以直接安全制作。

实际制作前，请务必查阅每个器件的数据手册和模块资料，并自行验证：

- 工作电压
- 逻辑电平
- 电流需求
- 接口类型
- 引脚复用
- 封装和接线方式
- 电源容量
- 散热需求
- 供应链和模块版本差异

涉及电机、继电器、4G 模块、高功耗灯带、外部电源、锂电池或其他较高风险硬件时，应额外关注过流保护、反接保护、共地、电平转换和电气安全。
